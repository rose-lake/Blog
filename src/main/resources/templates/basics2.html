Java Basics

String Objects

A String is an object, not a primitive data type. Yay, go String! As an object, String will have built-in behaviors.

Declaration

When you declare a string, you are creating an instance of the String class. Because strings are so heavily used, you're allowed to declare them in a similar syntax as we use with primitive data types. However, this is offered just as a convenience. What's really happening is we are declaring a new instance of the String type. Both declarations below are correct:

String hello = new String("Hello, World!");
String helloAgain = "Hello again, World!";

It is, however, "more correct" to use the full class-type declaration, this helps you remember what you're dealing with! ;)

String Constant Pool

String is a Final Class. Once created it cannot be altered. It is immutable. The JVM creates a memory location especially for Strings called the String Constant Pool. That's why Strings can be initialized without the new keyword. A string reference can be overwritten but that does not delete the original content... 

String h1 = "hello";
h1 = "this" + "world";

In this example code, original reference link to the String "hello" is broken. That string is still hanging around in memory with no handle...!

Conversely, if you create the exact same string over and over, Java will only have one version of it with different references pointing to it...

String h1 = "hello";
String h2 = "hello";
String h3 = "hello";

All three of these references point to the same string in the String Constant Pool.

String as char[]

A Java String is a sequence of single characters, it is basically a char[] array. You can declare a string from a character array, by passing the character array in to the String constructor. You can also convert a string back to its character array by using the .toCharArray() string method.

Escape Sequences

Any time you wish to include any of these special characters, such as a 'tab' or a 'new line', you must use the escape sequence in order for it to be rendered properly by the String class.

  • \t tab
  • \b backspace
  • \n new line
  • \r carriage return Note: Windows uses \r\n for a new line.
  • \f formfeed
  • \\ backslash
  • \" double-quote
  • \' single quote
Concatenating using +

The infix + operator has clearly been overridden in multiple ways to be able to accomplish String concatenation and also easily accept and render ints or other types and convert them (probably using their toString() methods) for display within a System.out.println() statement.

NOTE: Chars vs Strings

Interesting that using the + operator on Characters does not String-wise concatenate them. It prints/returns a number. Even though printing each character by itself will print/return the character as a viewable letter.
